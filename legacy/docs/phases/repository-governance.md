# Phase 0.5 — Repository Governance

## Goal

Introduce lightweight governance before the repository grows further.

## What changed

- the main branch convention was formalized
- a future integration branch convention was defined
- the repository license was added
- a publication guide was created
- course materials and local-only assets were explicitly kept outside version control

## Why this phase mattered

This phase made the repository easier to maintain and safer to evolve. It also separated reusable project assets from local or educational material that should not become part of the public codebase.

## Key artifacts

- `LICENSE`
- `docs/PUBLICATION_GUIDE.md`

## Closure

Phase 0.5 is complete because the repository gained explicit governance rules before deeper architectural and product work accelerated.

## Transition to the next phase

With governance in place, the next milestone focused on the user-facing product baseline.
