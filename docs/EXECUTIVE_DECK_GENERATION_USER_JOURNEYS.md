# Executive Deck Generation — user journeys

## Objective

Document the capability's main user journeys.

---

## Journey 1 — Benchmark & Eval Executive Review Deck

1. the user opens the benchmark/eval workflow
2. the product consolidates aggregated signals
3. the user triggers deck generation
4. the deck is rendered and downloaded

---

## Journey 2 — Document Review Deck

1. the user analyzes a document
2. the product generates findings/risks/recommendation
3. the user triggers the review deck
4. the deck becomes an executive handoff artifact

---

## Journey 3 — Policy / Contract Comparison Deck

1. the user compares documents
2. the product highlights differences and impacts
3. the user generates a comparison/decision deck

---

## Journey 4 — Candidate Review Deck

1. the CV is analyzed
2. strengths/gaps/recommendation are structured
3. the user generates the candidate's executive deck
