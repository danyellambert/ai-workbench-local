# Executive Deck Generation — user journeys

## Objetivo

Documentar jornadas principais da capability.

---

## Jornada 1 — Benchmark & Eval Executive Review Deck

1. usuário abre fluxo de benchmark/eval
2. produto consolida sinais agregados
3. usuário aciona geração do deck
4. deck é renderizado e baixado

---

## Jornada 2 — Document Review Deck

1. usuário analisa documento
2. produto gera findings/riscos/recomendação
3. usuário aciona deck de review
4. deck vira handoff executivo

---

## Jornada 3 — Policy / Contract Comparison Deck

1. usuário compara documentos
2. produto destaca diferenças e impactos
3. usuário gera deck de comparison/decision

---

## Jornada 4 — Candidate Review Deck

1. CV é analisado
2. strengths/gaps/recommendation são estruturados
3. usuário gera deck executivo do candidato
